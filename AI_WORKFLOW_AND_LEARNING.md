# NeuroNest AI Workflow And Learning

## Where Live Data Comes From

Indoor room data is generated by `simulator/simulator.py`.

Every few seconds it publishes MQTT messages like:

```text
home/living_room/sensor/all
home/kitchen/sensor/all
home/bathroom/sensor/all
```

Those messages contain:

```text
temperature, humidity, motion, smoke, light, timestamp, room
```

Open-Meteo is not the source of indoor live data. It is used by the agent only as outside weather context.

## Decision Flow

```text
Simulator sensor data
  -> MQTT
  -> AI Agent
  -> ContextAnalyzer
  -> ContextEnricher
  -> DecisionEngine
  -> PolicyManager
  -> MQTT device command
  -> Backend state + InfluxDB logs
  -> React dashboard
```

## Decision Priority

The decision engine now works in this order:

1. Safety rules
   - Smoke and very high humidity take priority.
2. Learned user preferences
   - Manual dashboard choices are stored as high-priority rules.
3. Generic ML model
   - The pretrained model from the larger dataset gives baseline behavior.
4. Gemini / LLM
   - Used for complex context when available.
5. Heuristics
   - Fallback rules for comfort, energy, and lighting.
6. Guards
   - Empty-room guard prevents lights turning on in unoccupied rooms.
   - Daylight guard prevents lights turning on during bright daytime unless the user has learned that preference.

## How Personalization Works

When the user manually toggles a device in the dashboard, the frontend sends feedback:

```text
POST /api/rooms/{room}/feedback
```

The backend publishes:

```text
home/{room}/feedback
```

The agent records the feedback in:

```text
agent/user_learning.json
```

The learned rule includes:

```text
room, device, command, time_period, occupancy, light level
```

This is intentionally stronger than adding a few rows to the big training CSV. The pretrained model remains the general baseline, but user feedback becomes a high-priority overlay. You do not need to rerun the big training job for user preferences to affect the demo.

```text

```

## Essence

NeuroNest uses a pretrained smart-home model as a baseline, then personalizes behavior with high-priority learned rules from the user's manual choices and mood input.
